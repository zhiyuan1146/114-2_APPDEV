## 1. 什麼是物件導向程式設計（OOP）？

之前寫的程式都是「做什麼事」的流程，例如用 for 迴圈印出九九乘法表。OOP 換一個思維方式：先想「有哪些東西」，再想「這些東西能做什麼」。

生活類比：設計一個港口管理系統，你會先想到有哪些「東西」：船、碼頭、貨櫃、船員。每個東西有自己的特性和行為。這些「東西」就是物件。

### OOP 四大特性預覽

| 特性 | 一句話 | 本週學 | 後續 |
|------|-------|:-----:|------|
| 封裝 | 隱藏細節，只開放介面 | | W04 |
| 繼承 | 子類別繼承父類別 | | W04 |
| 多型 | 同名方法，不同行為 | | W05 |
| 抽象 | 定義規格不實作 | | W06 |

本週先學最基礎的「Package」和「類別與物件」，這是四大特性的基石。

## 2. Package（套件）

### 2.1 什麼是 Package？

Package 是 Java 用來組織和分類類別的資料夾結構。就像電腦裡的資料夾，把相關的檔案放在一起，避免名稱衝突，也方便管理。

生活類比：一間大公司有「人事部」「財務部」「研發部」，每個部門有自己的文件和人員。Package 就像這些部門，把相關的類別歸在一起。

### 2.2 為什麼需要 Package？

| 沒有 Package | 有 Package |
|-------------|-----------|
| 所有類別放在一起，容易混亂 | 類別按功能分門別類 |
| 兩個人寫了同名的 `Util` 類別會衝突 | `com.team1.Util` 和 `com.team2.Util` 不衝突 |
| 找不到想用的類別在哪裡 | 看 package 名稱就知道功能 |

### 2.3 宣告 Package

在 .java 檔案的第一行宣告所屬的 package：

```java
package com.ocean.model;    // 宣告這個類別屬於 com.ocean.model 套件

public class Ship {
    String name;
    // ...
}
```

檔案的存放位置必須和 package 對應：

```
src/
└── com/
    └── ocean/
        ├── model/
        │   ├── Ship.java        ← package com.ocean.model;
        │   └── MarineLife.java  ← package com.ocean.model;
        └── util/
            └── Calculator.java  ← package com.ocean.util;
```

### 2.4 匯入其他 Package 的類別

用 `import` 引入其他 package 的類別：

```java
package com.ocean.app;

import com.ocean.model.Ship;          // 匯入單一類別
import com.ocean.model.*;             // 匯入整個 package 的所有類別

public class Main {
    public static void main(String[] args) {
        Ship s = new Ship("遠洋號");   // 因為有 import，可以直接用
    }
}
```

### 2.5 Java 內建的常用 Package

Java 本身提供了很多 package，有些需要 import，有些不用：

| Package | 功能 | 需要 import？ | 範例 |
|---------|------|:----------:|------|
| `java.lang` | 基本類別 | 不需要，自動引入 | `String`、`System`、`Math` |
| `java.util` | 工具類別 | 需要 | `ArrayList`、`HashMap`、`Scanner` |
| `java.io` | 檔案輸入輸出 | 需要 | `File`、`BufferedReader` |
| `java.math` | 數學計算 | 需要 | `BigDecimal` |

```java
// java.lang 不需要 import，直接用
String name = "Hello";
System.out.println(name);
double pi = Math.PI;

// java.util 需要 import
import java.util.ArrayList;
import java.util.Scanner;

ArrayList<String> list = new ArrayList<>();
Scanner sc = new Scanner(System.in);
```

### 2.6 Package 命名慣例

| 規則 | 說明 | 範例 |
|------|------|------|
| 全小寫 | 不用大寫 | `com.ocean.model` ✅ |
| 反向網域名 | 避免和別人衝突 | `com.nkust.appdev` |
| 有意義的層級 | 按功能分類 | `model`、`util`、`view`、`controller` |

### 2.7 目前階段的簡化用法

本課程前 6 週的練習檔案比較簡單，暫時不需要宣告 package，直接寫類別即可。等 W07 進入 Android 開發後，Android Studio 會自動幫你建立 package 結構，例如：

```
com.example.myapp/
├── MainActivity.java
├── model/
│   └── MarineLife.java
└── adapter/
    └── MarineAdapter.java
```

到時候你已經理解 package 的觀念，就能看懂 Android 的專案結構了。

## 3. 類別（Class）與物件（Object）

**類別**是設計圖，**物件**是根據設計圖做出來的實體。

| 概念 | 類比 | Java |
|------|------|------|
| 類別 Class | 船的設計圖 | `class Ship { ... }` |
| 物件 Object | 根據設計圖造出的一艘船 | `Ship s = new Ship();` |

一份設計圖可以造出很多艘船，每艘船是獨立的物件，有自己的名字和狀態。

---

## 4. 定義類別

### 4.1 基本結構

```java
class Ship {
    // 屬性（Attributes）：這個東西「有什麼」
    String name;
    int capacity;
    double speed;

    // 方法（Methods）：這個東西「能做什麼」
    String info() {
        return name + "，載量 " + capacity + " 噸，速度 " + speed + " 節";
    }

    void sail() {
        System.out.println(name + " 正在航行");
    }
}
```

### 4.2 建立物件

用 `new` 關鍵字根據類別建立物件：

```java
Ship s1 = new Ship();       // 建立一個 Ship 物件
s1.name = "遠洋號";         // 設定屬性
s1.capacity = 5000;
s1.speed = 20.5;

Ship s2 = new Ship();       // 建立另一個 Ship 物件
s2.name = "海豐號";
s2.capacity = 3000;
s2.speed = 15.0;

System.out.println(s1.info());  // 遠洋號，載量 5000 噸，速度 20.5 節
System.out.println(s2.info());  // 海豐號，載量 3000 噸，速度 15.0 節
s1.sail();                      // 遠洋號 正在航行
```

s1 和 s2 是兩個獨立的物件，修改 s1 不會影響 s2。

## 5. 建構子（Constructor）

每次建立物件都要一行一行設定屬性很麻煩。建構子讓你在建立物件的同時就初始化屬性。

### 5.1 基本建構子

```java
class Ship {
    String name;
    int capacity;
    double speed;

    // 建構子：和類別同名，沒有回傳值
    Ship(String name, int capacity, double speed) {
        this.name = name;           // this 指向目前這個物件
        this.capacity = capacity;
        this.speed = speed;
    }

    String info() {
        return name + "，載量 " + capacity + " 噸，速度 " + speed + " 節";
    }
}
```

使用建構子建立物件：

```java
Ship s1 = new Ship("遠洋號", 5000, 20.5);   // 一行搞定
Ship s2 = new Ship("海豐號", 3000, 15.0);

System.out.println(s1.info());
System.out.println(s2.info());
```

### 5.2 this 關鍵字

`this` 指向「目前這個物件」。當參數名稱和屬性名稱相同時，用 `this` 區分：

```java
Ship(String name, int capacity, double speed) {
    this.name = name;         // this.name 是屬性，name 是參數
    this.capacity = capacity;
    this.speed = speed;
}
```

### 5.3 建構子多載

一個類別可以有多個建構子，參數不同：

```java
class Ship {
    String name;
    int capacity;
    double speed;

    // 建構子 1：完整參數
    Ship(String name, int capacity, double speed) {
        this.name = name;
        this.capacity = capacity;
        this.speed = speed;
    }

    // 建構子 2：只給名字，其他用預設值
    Ship(String name) {
        this.name = name;
        this.capacity = 1000;    // 預設值
        this.speed = 10.0;       // 預設值
    }

    String info() {
        return name + "，載量 " + capacity + " 噸，速度 " + speed + " 節";
    }
}
```

```java
Ship s1 = new Ship("遠洋號", 5000, 20.5);  // 用建構子 1
Ship s2 = new Ship("小漁船");               // 用建構子 2

System.out.println(s1.info());  // 遠洋號，載量 5000 噸，速度 20.5 節
System.out.println(s2.info());  // 小漁船，載量 1000 噸，速度 10.0 節
```

## 6. 方法（Methods）

方法定義物件「能做什麼」。

### 6.1 有回傳值的方法

```java
class Rectangle {
    double width;
    double height;

    Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }

    // 回傳 double
    double area() {
        return width * height;
    }

    // 回傳 String
    String describe() {
        return "長方形 " + width + " x " + height + "，面積 = " + area();
    }
}
```

### 6.2 有參數的方法

```java
class Ship {
    String name;
    double fuelLevel;

    Ship(String name) {
        this.name = name;
        this.fuelLevel = 100.0;
    }

    // 有參數、沒回傳值
    void sail(int km) {
        double fuelUsed = km * 0.5;
        fuelLevel -= fuelUsed;
        System.out.println(name + " 航行 " + km + " 公里，剩餘油量：" + fuelLevel);
    }

    // 有參數、有回傳值
    double estimateFuel(int km) {
        return km * 0.5;
    }
}
```

```java
Ship s = new Ship("遠洋號");
s.sail(50);                                    // 遠洋號 航行 50 公里，剩餘油量：75.0
System.out.println("預估油耗：" + s.estimateFuel(100));  // 預估油耗：50.0
```

## 7. 完整範例：海洋生物

把以上觀念整合，建立一個海洋生物的類別：

```java
class MarineLife {
    String name;
    String species;
    String habitat;
    double weight;

    // 建構子
    MarineLife(String name, String species, String habitat, double weight) {
        this.name = name;
        this.species = species;
        this.habitat = habitat;
        this.weight = weight;
    }

    // 簡單建構子
    MarineLife(String name, String species) {
        this.name = name;
        this.species = species;
        this.habitat = "未知";
        this.weight = 0;
    }

    // 方法
    String info() {
        return name + "（" + species + "），棲息地：" + habitat + "，體重：" + weight + " kg";
    }

    String move() {
        return name + " 在水中移動";
    }

    String eat(String food) {
        return name + " 正在吃 " + food;
    }
}

public class MarineDemo {
    public static void main(String[] args) {
        MarineLife nemo = new MarineLife("尼莫", "小丑魚", "珊瑚礁", 0.1);
        MarineLife whale = new MarineLife("藍藍", "藍鯨", "太平洋", 150000);
        MarineLife unknown = new MarineLife("神秘生物", "未知種");

        System.out.println(nemo.info());
        System.out.println(nemo.move());
        System.out.println(nemo.eat("海葵"));
        System.out.println();
        System.out.println(whale.info());
        System.out.println(whale.move());
        System.out.println(whale.eat("磷蝦"));
        System.out.println();
        System.out.println(unknown.info());
    }
}
```

輸出：
```
尼莫（小丑魚），棲息地：珊瑚礁，體重：0.1 kg
尼莫 在水中移動
尼莫 正在吃 海葵

藍藍（藍鯨），棲息地：太平洋，體重：150000.0 kg
藍藍 在水中移動
藍藍 正在吃 磷蝦

神秘生物（未知種），棲息地：未知，體重：0.0 kg
```

## 8. 常見錯誤

| 錯誤 | 原因 | 修正 |
|------|------|------|
| `NullPointerException` | 物件沒有用 new 建立就使用 | 確保 `new` 之後才呼叫方法 |
| 建構子有 `void` | 建構子不需要寫回傳型別 | 拿掉 `void` |
| `this` 忘記寫 | 參數名和屬性名一樣時沒用 this 區分 | 加上 `this.` |
| 一個檔案多個 public class | 只有和檔名同名的 class 可以加 public | 其他 class 拿掉 public |
| import 忘記寫 | 使用 java.util 的類別沒有 import | 加上對應的 import |
